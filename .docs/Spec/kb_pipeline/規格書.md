# KB Pipeline 規格書

## 概述

`kb_pipeline` 是 AIBox 知識庫的離線處理引擎，負責將上傳的文件轉換為可檢索的向量與知識圖譜。所有元件皆為獨立的 Python 模組，可被 Celery、非同步端點或其他 Python 服務直接呼叫。

## 架構

```
kb_pipeline/
├── __init__.py       統一匯出
├── chunker.py        文字分塊（無依賴）
├── embedder.py       Ollama 向量化
├── qdrant_ops.py    Qdrant 向量儲存 CRUD
├── arango_ops.py    ArangoDB 文件與圖譜操作
├── graph.py         LLM 知識圖譜抽取
└── pipeline.py       向量化 + 圖譜抽取協調器
```

## 依賴關係

```
pipeline.py
├── chunker.py    (無依賴)
├── embedder.py   (httpx)
├── qdrant_ops.py (httpx)
├── arango_ops.py (httpx, pathlib)
└── graph.py     (httpx, json)
```

## 各模組規格

### chunker.py

無狀態純函式，無外部依賴。

```python
from kb_pipeline import chunk_text

chunks: list[str] = chunk_text(
    text: str,
    chunk_size: int = 500,      # 每塊字元數
    chunk_overlap: int = 100,    # 重疊字元數
)
```

**行為**：
- 依序滑動視窗，以 `chunk_size - chunk_overlap` 為步進
- 過濾空塊
- 對中文（無空白分隔）依字元切分

**測試**：
```python
chunk_text("")        → []
chunk_text("abc")     → ["abc"]  (長度 < 500)
chunk_text("a" * 600) → ["a"*500, "a"*100 + "a"*400]  (有重疊)
```

### embedder.py

Ollama 向量化客戶端，支援單筆與批量。

```python
from kb_pipeline import Embedder

emb = Embedder(
    base_url: str | None = None,   # 預設 OLLAMA_BASE_URL
    model: str | None = None,       # 預設 EMBEDDING_MODEL
    timeout: float = 120.0,
)

vec: list[float] = emb.embed("文字")
vecs: list[list[float]] = emb.embed_batch(["文字1", "文字2"])
```

**行為**：
- 呼叫 `POST {base_url}/api/embed`
- 單筆：`{"model": model, "input": text}`
- 批量：`{"model": model, "input": [text1, text2]}`
- 失敗時回傳空陣列，不拋例外

**環境變數**：

| 變數 | 預設值 |
|------|--------|
| `OLLAMA_BASE_URL` | `http://localhost:11434` |
| `EMBEDDING_MODEL` | `bge-m3:latest` |

### qdrant_ops.py

Qdrant 向量儲存的 CRUD 封裝。

```python
from kb_pipeline import QdrantStore
from dataclasses import dataclass

@dataclass
class Point:
    id: int
    vector: list[float]
    file_id: str
    root_id: str
    chunk_index: int
    text: str
    text_full: str

qdrant = QdrantStore(
    url: str | None = None,   # 預設 QDRANT_URL
    timeout: float = 60.0,
)

qdrant.ensure_collection("knowledge_kb2")           # 確保 collection 存在
qdrant.upsert("knowledge_kb2", [Point(...)])       # 寫入 points
results: list[dict] = qdrant.search(vec, limit=5)    # 向量相似性檢索
count: int = qdrant.count("knowledge_kb2")         # 統計點數
```

**Collection 參數**：
- 向量維度：1024（`bge-m3`）
- 距離：`Cosine`
- `indexing_threshold`：10000

**環境變數**：

| 變數 | 預設值 |
|------|--------|
| `QDRANT_URL` | `http://localhost:6333` |

### arango_ops.py

ArangoDB 文件操作，支援知識庫文件狀態與圖譜寫入。

```python
from kb_pipeline import ArangoOps

arango = ArangoOps(
    url: str | None = None,
    db: str | None = None,
    user: str | None = None,
    password: str | None = None,
)

arango.update_status(file_id, vector_status="completed")
arango.update_status(file_id, graph_status="processing")
arango.update_status(file_id, vector_status="failed", graph_status="pending")

text: str | None = arango.read_file(file_id)        # 讀取磁碟上的原始檔案
chunks: list[str] = arango.read_file_chunks(file_id)  # 讀取知識塊集合
doc: dict | None = arango.get_file(file_id)         # 讀取文件本身

arango.upsert_graph(file_id, nodes, edges)           # 寫入圖譜節點與邊
```

**Collection/Graph 寫入**：
- 節點寫入 `knowledge_graphs`
- 邊寫入 `knowledge_graph_edges`
- 自動產生 `_key`： `{file_id}_node_{i}`, `{file_id}_edge_{i}`

**環境變數**：

| 變數 | 預設值 |
|------|--------|
| `ARANGO_URL` | `http://localhost:8529` |
| `ARANGO_DATABASE` | `abc_desktop` |
| `ARANGO_USER` | `root` |
| `ARANGO_PASSWORD` | `abc_desktop_2026` |

### graph.py

使用 LLM（Ollama）從文字抽取知識圖譜。

```python
from kb_pipeline import GraphExtractor

extractor = GraphExtractor(
    ollama_url: str | None = None,
    model: str | None = None,       # 預設 OLLAMA_LLM_MODEL
    timeout: float = 120.0,
)

entities: list[dict], relations: list[dict] = extractor.extract(text)
```

**回傳格式**：
```python
{
    "entities": [
        {"entity": "AIBox", "entity_type": "技術", "description": "知識管理系統"},
        {"entity": "RAG", "entity_type": "概念", "description": "檢索增強生成"},
    ],
    "relations": [
        {"source": "AIBox", "target": "RAG", "relation": "使用"},
    ]
}
```

**Prompt 策略**：
- 溫度 0.1（確定性輸出）
- 最大生成 512 tokens
- 失敗時回傳空陣列，不拋例外

**環境變數**：

| 變數 | 預設值 |
|------|--------|
| `OLLAMA_BASE_URL` | `http://localhost:11434` |
| `OLLAMA_LLM_MODEL` | `qwen2.5-coder:7b` |

### pipeline.py

協調器，組合以上所有元件，提供 `vectorize` 與 `extract_graph` 兩個核心方法。

```python
from kb_pipeline import Pipeline

p = Pipeline(
    arango: ArangoOps | None = None,
    embedder: Embedder | None = None,
    qdrant: QdrantStore | None = None,
    graph: GraphExtractor | None = None,
)

result: VectorizeResult = p.vectorize(file_id, local_path, root_id)
result: GraphResult = p.extract_graph(file_id, local_path)
```

**VectorizeResult**：
```python
{"chunks": 3, "status": "completed"}
{"chunks": 0, "status": "no_content"}
{"chunks": 0, "status": "embedding_failed"}
{"chunks": 0, "status": "failed"}
```

**GraphResult**：
```python
{"entities": 5, "relations": 3, "status": "completed"}
{"entities": 0, "relations": 0, "status": "no_content"}
{"entities": 0, "relations": 0, "status": "failed"}
```

**文字來源優先順序**：
1. `ArangoOps.read_file()` — 讀取磁碟上的原始檔案
2. `ArangoOps.read_file_chunks()` — 讀取 `knowledge_chunks` 集合
3. `Path(local_path)` — 直接讀取路徑

**狀態更新時機**：
- `vectorize`：開始 → `processing`，結束 → `completed` 或 `failed`
- `extract_graph`：開始 → `processing`，結束 → `completed` 或 `failed`

## HTTP 端點（knowledge_agent）

所有端點位於 `knowledge_agent` 服務（Port 8007），由 Rust API 在檔案上傳後觸發。

### POST /pipeline/trigger

舊版端點，支援 task type 參數。

```json
POST /pipeline/trigger
{
    "task": "process_file",   // process_file | vectorize | graph
    "file_id": "uuid",
    "local_path": "/path/to/file.txt",
    "root_id": "kb2"
}
```

### POST /pipeline/vector

獨立觸發向量化。自動從 ArangoDB 取得 `local_path`。

```
POST /pipeline/vector?file_id={file_id}&root_id={root_id}
```

### POST /pipeline/graph

獨立觸發圖譜抽取。自動從 ArangoDB 取得 `local_path`。

```
POST /pipeline/graph?file_id={file_id}
```

## Celery Tasks

| Task | 參數 | 作用 |
|------|------|------|
| `process_file_task` | `file_id, local_path, root_id` | 向量化 + 圖譜 |
| `vectorize_task` | `file_id, local_path, root_id` | 僅向量化 |
| `graph_task` | `file_id, local_path` | 僅圖譜 |

所有 tasks 皆有 `max_retries=3`，重試間隔 30 秒。

## 與 Rust API 的整合

Rust API 在 `/api/v1/knowledge/roots/{root_id}/files/upload` 接收上傳後：

1. 將檔案寫入磁碟（`{cwd}/data/uploads/{root_id}/{file_id}.{ext}`）
2. 寫入 ArangoDB `knowledge_files` 記錄（包含 `local_path`、`s3_path`、`vector_status: pending`、`graph_status: pending`）
3. 備份到 SeaweedFS ai-box 叢集
4. 呼叫 `POST http://knowledge_agent:8007/pipeline/trigger` 將 Celery task 排入 Redis 佇列

## 錯誤處理策略

| 情境 | 策略 |
|------|------|
| Ollama 無回應 | 單次 embed 失敗 → 回傳空 → 整體 `embedding_failed` |
| Qdrant upsert 失敗 | 拋例外 → Celery retry (3次, 30s) |
| LLM 回應非 JSON | `json.loads` 失敗 → 回傳 `[]` |
| 檔案不存在 | `Path.read_text` 失敗 → `read_file_chunks` → 都失敗則 `failed` |
| ArangoDB 更新失敗 | Celery retry |
