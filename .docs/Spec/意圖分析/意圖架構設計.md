# Orchestrator 意圖分析與結構化處理架構設計文件（v1.0）


## 1. 文件目的（Purpose）

本文件旨在定義企業級 AI Orchestrator 中「意圖分析（Intent Analysis）」與「結構化資料轉換（Structured Representation）」的設計架構，確保：

* 多輪對話可控
* 任務可執行（Actionable）
* 系統可維運（Maintainable）
* 模型可持續優化（Evolvable）

---

## 2. 系統定位（System Positioning）

本模組位於 AI Orchestrator 核心層，負責將：

```text
自然語言（User Input）
→ 意圖識別（Intent）
→ 結構化資料（Structured Data）
→ 可執行行為（Action）
```

---

## 3. 整體架構（High-Level Architecture）

```text
[User Input]
     ↓
[LLM Intent Parser]
     ↓
[Intent Dictionary（Registry）]
     ↓
[Entity Extraction]
     ↓
[Context Manager]
     ↓
[State Machine Controller]
     ↓
[Skill Mapping]
     ↓
[Tool / API Execution]
```

---

## 4. 核心模組說明（Core Components）

---

### 4.1 Intent Parser（意圖解析器）

**功能：**

* 使用 LLM 進行語意理解
* 輸出標準化 JSON

**輸出格式：**

```json
{
  "intent": {},
  "entities": {},
  "confidence": 0.0
}
```

---

### 4.2 Intent Dictionary（意圖字典）

**功能：**

* 定義所有可支援的業務意圖
* 提供分類與執行標準

**設計原則：**

* 每個 Intent 對應一個業務行為
* 與 API / Service 綁定
* 支援版本控管

**範例：**

```json
{
  "intent_id": "INV-QUERY-001",
  "domain": "inventory",
  "type": "query",
  "name": "check_stock",
  "required_entities": ["item_code"],
  "action": {
    "service": "inventory_service",
    "method": "get_stock"
  }
}
```

---

### 4.3 Entity Extraction（實體抽取）

**功能：**

* 從語句中提取關鍵參數
* 對齊資料庫 Schema

**設計原則：**

* Entity 必須可直接映射 DB
* 避免自然語言格式

---

### 4.4 Context Manager（上下文管理）

**功能：**

* 管理多輪對話狀態
* 控制流程決策

**結構：**

```json
{
  "source": "user",
  "priority": "normal",
  "time_range": null
}
```

---

### 4.5 State Machine（狀態機控制）

**功能：**

* 控制任務執行流程
* 處理不完整輸入（clarification）

**狀態範例：**

```text
INIT
 → INTENT_PARSED
 → NEED_CLARIFICATION
 → READY
 → EXECUTING
 → COMPLETED
```

---

### 4.6 Skill Mapping（技能映射）

**功能：**

* 將 Intent 映射為可執行邏輯

```text
Intent → Skill → Tool/API
```

---

### 4.7 Tool / API Layer（執行層）

**功能：**

* 呼叫實際服務（ERP / DB / API）

---

## 5. 結構化資料設計（Canonical Schema）

---

### 5.1 標準輸出格式

```json
{
  "intent": {
    "domain": "inventory",
    "type": "query",
    "name": "check_stock",
    "intent_id": "INV-QUERY-001",
    "version": "1.0"
  },
  "entities": {
    "item_code": "A123",
    "warehouse": null
  },
  "context": {
    "source": "user",
    "priority": "normal"
  },
  "action": {
    "service": "inventory_service",
    "method": "get_stock",
    "confidence": 0.87
  },
  "state": {
    "status": "need_clarification"
  },
  "clarification": {
    "required": true,
    "missing_fields": ["warehouse"]
  }
}
```

---

## 6. Intent 維護策略（Governance）

---

### 6.1 命名規範

```text
{DOMAIN}-{TYPE}-{NUMBER}
```

例：

* INV-QUERY-001
* FIN-ANALYSIS-002

---

### 6.2 分類控制

| 層級   | 說明     |
| ------ | -------- |
| domain | 業務領域 |
| type   | 行為類型 |
| name   | 具體動作 |

---

### 6.3 數量控制

* 每個 domain：10–20 個 intent
* 全系統：< 50 個（初期）

---

## 7. Entity 管理（Entity Registry）

---

### 7.1 定義

```json
{
  "item_code": {
    "type": "string",
    "source_table": "item_master",
    "required": true
  }
}
```

---

### 7.2 原則

* 與 DB Schema 對齊
* 支援驗證（validation）
* 支援預設值與補值

---
