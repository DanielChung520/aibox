## 前置準備

### 環境需求

| 工具        | 版本需求  | 檢查指令                 |
| ----------- | --------- | ------------------------ |
| Node.js     | 18+       | `node -v`                |
| Rust        | 1.70+     | `rustc --version`        |
| Tauri CLI   | 2.x       | `npx tauri --version`    |
| Xcode CLT   | 最新      | `xcode-select -p`        |

### Rust Target 確認

```bash
# 確認已安裝兩個 macOS target
rustup target list --installed
# 應包含：
#   aarch64-apple-darwin
#   x86_64-apple-darwin

# 若缺少，安裝：
rustup target add aarch64-apple-darwin
rustup target add x86_64-apple-darwin
```

### Updater 簽署金鑰（必要）

自動更新功能依賴 **minisign（Ed25519）** 簽署機制。每次打包時，CI 會用私鑰對更新包簽名；使用者端 App 啟動時，Updater 插件會用公鑰驗證簽名，確保更新包未被篡改。

> **首次設定只需執行一次。** 若已產生過金鑰，跳過此步驟。

#### 1. 產生金鑰對

```bash
cd src-tauri
npx tauri signer generate --ci
```

輸出範例：

```
Writing private key to ~/.tauri/abc-desktop.key
Writing public key to  ~/.tauri/abc-desktop.pub

Public key: dW50cnVzdGVkIGNvbW1lbnQ6IG1pbmlzaWduIHB1YmxpYyBrZXk6IDg5MzI4OTIwNkJBOENEOUUKUldTZXphaHJJSWt5aVJNUVRicWFSSzV4QWJHM1VpS2M0RGNxL05IbFpoUlZUOWpmL01YK1M3OXUK

[to.GitHub.Actions]
TAURI_SIGNING_PRIVATE_KEY=-----BEGIN Tauri Private Key-----
MIIBUjA9...
-----END Tauri Private Key-----
TAURI_SIGNING_PRIVATE_KEY_PASSWORD=
```

#### 2. 更新公鑰到 `tauri.conf.json`

```bash
# 複製新公鑰
cat ~/.tauri/*.pub
```

將 `src-tauri/tauri.conf.json` 的 `plugins.updater.pubkey` 換成新公鑰。

#### 3. 設定 GitHub Secrets

到 **GitHub → Settings → Secrets and variables → Actions → New repository secret**，新增兩個 secret：

| Secret 名稱 | 值 |
|---|---|
| `TAURI_SIGNING_PRIVATE_KEY` | `~/.tauri/*.key` 的完整內容（包含 `-----BEGIN...` 行） |
| `TAURI_SIGNING_PRIVATE_KEY_PASSWORD` | 留空（或填入你產生金鑰時設定的密碼） |

#### 4. 驗證本地打包（含簽名）

```bash
# 設定環境變數後打包
export TAURI_SIGNING_PRIVATE_KEY="$(cat ~/.tauri/*.key)"
export TAURI_SIGNING_PRIVATE_KEY_PASSWORD=""
npm run tauri:build:mac-arm
```

確認 `target/aarch64-apple-darwin/release/bundle/macos/` 下有 `.sig` 簽名檔：

```
ABC管理系统.app.tar.gz
ABC管理系统.app.tar.gz.sig   ← 有這個代表簽名成功
```

## Step 2：重新產生 App Icon

若 icon 有更新，必須從來源圖（`src-tauri/icons/1024x1024.png`，建議 1024×1024 px 以上）重新產生所有尺寸。

### 2.1 用 sharp 產生所有 PNG 尺寸

`npx tauri icon` 只產生部分尺寸（32, 64, 128, 128@2x, icon.icns, icon.ico, icon.png），不會覆蓋 `@2x` 高解析度和大尺寸 PNG。需先用 `sharp` 補齊：

```bash
node -e "
const sharp = require('sharp');
const path = require('path');
const src = 'src-tauri/icons/1024x1024.png';

const sizes = [
  { name: '1024x1024@2x.png', size: 2048 },
  { name: '512x512.png', size: 512 },
  { name: '512x512@2x.png', size: 1024 },
  { name: '256x256.png', size: 256 },
  { name: '256x256@2x.png', size: 512 },
  { name: '128x128.png', size: 128 },
  { name: '128x128@2x.png', size: 256 },
  { name: '64x64.png', size: 64 },
  { name: '64x64@2x.png', size: 128 },
  { name: '32x32.png', size: 32 },
  { name: '32x32@2x.png', size: 64 },
  { name: '16x16.png', size: 16 },
  { name: '16x16@2x.png', size: 32 },
  { name: 'icon.png', size: 512 },
];

(async () => {
  for (const s of sizes) {
    await sharp(src).resize(s.size, s.size).png()
      .toFile(path.join('src-tauri/icons', s.name));
    console.log('Generated:', s.name);
  }
})();
"
```

### 2.2 用 tauri icon 產生 .icns / .ico 及平台 icon

```bash
npx tauri icon src-tauri/icons/1024x1024.png
```

此指令會產生：
- `icon.icns`（macOS）
- `icon.ico`（Windows）
- `Square*.png`（Windows Store）
- `ios/` 及 `android/` 目錄下的平台 icon

### 2.3 驗證所有 icon 都已更新

```bash
# 所有 icon 檔案的時間戳應一致（除 1024x1024.png 原始來源外）
ls -la src-tauri/icons/*.png src-tauri/icons/*.icns src-tauri/icons/*.ico
```

## Step 6：發佈到 abc.k84.org

### 6.1 本機部署（abc.k84.org 指向本機）

```bash
# 複製下載頁
cp site/index.html ~/abc-site/index.html

# 複製 DMG（將 {version} 替換為實際版本號）
cp "target/aarch64-apple-darwin/release/bundle/dmg/ABC管理系统_{version}_aarch64.dmg" ~/abc-site/downloads/
cp "target/x86_64-apple-darwin/release/bundle/dmg/ABC管理系统_{version}_x64.dmg" ~/abc-site/downloads/
```

### 6.2 啟動 Web Server

Python 的 `http.server` 對中文檔名有 URL decode 問題，使用自訂 handler：

```bash
# 使用 .tmp/serve.py（已包含 UTF-8 修正）
pkill -f 'python3.*serve.py' 2>/dev/null
sleep 1
nohup python3 .tmp/serve.py > /dev/null 2>&1 &
```

`.tmp/serve.py` 內容：

```python
import http.server, socketserver, os, urllib.parse

os.chdir(os.path.expanduser('~/abc-site'))

class UTF8Handler(http.server.SimpleHTTPRequestHandler):
    def translate_path(self, path):
        path = urllib.parse.unquote(path)
        return super().translate_path(path)

with socketserver.TCPServer(('', 6000), UTF8Handler) as httpd:
    httpd.serve_forever()
```

### 6.3 遠端部署（SSH）

若 abc.k84.org 指向遠端伺服器：

```bash
./deploy-site.sh user@your-server
```

## 附記：Updater 更新機制說明

### 工作原理

```
┌──────────────────────────────┐
│  使用者電腦上的 ABC Desktop   │
│                               │
│  App 啟動 → AppUpdater 元件   │
│    ↓ 每 4 小時自動檢查一次    │
│    GET https://github.com/   │
│      DanielChung520/aibox/   │
│      releases/latest/download│
│      /latest.json            │
│    ↓                         │
│  比對版本號                   │
│    ├── v1.0.2 < v1.0.3      │
│    │   → 彈窗提示使用者       │
│    │   → 下載新安裝包        │
│    │   → 重啟應用             │
│    └── v1.0.2 == v1.0.3      │
│        → 什麼都不做           │
└──────────────────────────────┘
```

### 完整流程（版本更新）

| 順序 | 動作 | 負責方 |
|------|------|--------|
| 1 | 開發者推送 `git tag v1.0.3` | 開發者 |
| 2 | GitHub Actions 觸發 CI | GitHub |
| 3 | 編譯前端、編譯 Rust 殼層 | CI |
| 4 | 對每個平台的安裝包簽名，產生 `.sig` | CI |
| 5 | 產生 `latest.json`（含簽名、URL、SHA256） | CI |
| 6 | 上傳所有檔案到 GitHub Release | CI |
| 7 | 使用者下次開啟 App → 偵測到新版本 | App |
| 8 | 提示「發現新版本 v1.0.3」→ 下載 → 重啟 | App |

### 使用者視角

```
首次安裝 v1.0.2（DMG）
  ↓
正常使用
  ↓
某天開啟 App → 偵測到 v1.0.3
  → 彈窗：「發現新版本，是否立即更新？」
  → 按「立即更新」
  → 背景下載（約 12 MB）
  → 自動重啟 → 已是最後版本
  （不需要重新下載 DMG，不需要手動安裝）
```

### latest.json 格式（Tauri 2.x）

```json
{
  "version": "1.0.3",
  "notes": "Release v1.0.3",
  "pub_date": "2026-03-29T12:00:00Z",
  "platforms": {
    "darwin-aarch64": {
      "signature": "<minisign base64 signature>",
      "url": "https://github.com/DanielChung520/aibox/releases/download/v1.0.3/ABC管理系统_1.0.3_aarch64.app.tar.gz",
      "size": 12582912,
      "checksum": "sha256:abc123..."
    },
    "darwin-x86_64": { ... },
    "windows-x86_64": { ... }
  }
}
```

### 更新時機

- **App 啟動時**：自動檢查一次
- **每 4 小時**：背景定期檢查（`AppUpdater.tsx` 中的 `CHECK_INTERVAL_MS`）
- **手動檢查**：可擴充「檢查更新」按鈕觸發 `check()`

## 附記：Windows 打包（Cloudflare CI）

### 為什麼不能在 macOS 打包 Windows？

Tauri 的 Windows 安裝檔（NSIS / MSI）依賴 Windows 原生工具鏈：

| 安裝格式 | 依賴工具          | macOS 可用？ |
| -------- | ----------------- | ------------ |
| NSIS     | NSIS 編譯器       | ❌            |
| MSI      | WiX Toolset       | ❌            |

Rust 本身可以 cross-compile 到 `x86_64-pc-windows-msvc`，但 Tauri 的 **bundle 步驟**（將 `.exe` 包成安裝檔）無法在非 Windows 環境執行。

### 方案：使用 Cloudflare CI / GitHub Actions

在 CI 環境中配置 Windows runner 來打包：

```yaml
# .github/workflows/build.yml（參考）
name: Build & Release

on:
  push:
    tags: ['v*']

jobs:
  build-windows:
    runs-on: windows-latest
    steps:
      - uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: 20

      - name: Setup Rust
        uses: dtolnay/rust-toolchain@stable

      - name: Install dependencies
        run: npm ci

      - name: Build Tauri (Windows)
        run: npx tauri build --target x86_64-pc-windows-msvc

      - name: Upload artifacts
        uses: actions/upload-artifact@v4
        with:
          name: windows-installers
          path: |
            target/x86_64-pc-windows-msvc/release/bundle/nsis/*.exe
            target/x86_64-pc-windows-msvc/release/bundle/msi/*.msi
```

### Windows 打包產出

```
target/x86_64-pc-windows-msvc/release/bundle/
├── nsis/ABC管理系统_{version}_x64-setup.exe    # NSIS 安裝檔
└── msi/ABC管理系统_{version}_x64.msi           # MSI 安裝檔
```

### tauri.conf.json 中的 Windows 配置

```json
{
  "bundle": {
    "targets": ["app", "dmg", "nsis", "msi"],
    "windows": {
      "certificateThumbprint": null,
      "digestAlgorithm": "sha256",
      "nsis": {
        "displayLanguageSelector": true,
        "languages": ["SimpChinese", "English"],
        "installMode": "both"
      }
    }
  }
}
```

### Windows 安裝檔整合至下載頁

Windows 安裝檔打包完成後：

1. 從 CI 下載產出的 `.exe` / `.msi`
2. 複製到 `~/abc-site/downloads/`
3. 更新 `site/index.html` 的 Windows 下載卡片（將 `btn-disabled` 改為 `btn`，加上 `href`）

## 常見問題

### Q：打包時報 `TAURI_SIGNING_PRIVATE_KEY` 錯誤

確認已完成[金鑰設定](#前置準備)，且 GitHub Secrets 中 `TAURI_SIGNING_PRIVATE_KEY` 已正確設定。若暫不需要自動更新，可將 `tauri.conf.json` 的 `createUpdaterArtifacts` 設為 `"false"` 再打包。

### Q：Updater 提示「簽名驗證失敗」

可能原因：
1. 公鑰和私鑰不匹配 → 確認 `tauri.conf.json` 的 `pubkey` 與產生金鑰時的公鑰一致
2. GitHub Secrets 中的私鑰格式錯誤 → 確認包含 `-----BEGIN Tauri Private Key-----` 和 `-----END Tauri Private Key-----`

### Q：DMG 下載返回 404

Python `http.server` 對中文 URL 的 percent-decode 有問題。改用 `.tmp/serve.py` 的自訂 handler，或改用 nginx / caddy 等正式 web server。

### Q：打包後 app icon 沒更新

`npx tauri icon` 只產生部分尺寸（32, 64, 128, 128@2x, icon.icns, icon.ico, icon.png），不會覆蓋 `@2x` 高解析度和大尺寸 PNG。需要先用 `sharp` 從來源圖重新產生所有尺寸，再跑 `tauri icon`。

### Q：macOS 首次打開被 Gatekeeper 阻擋

應用未經 Apple 簽署，使用者需右鍵 → 打開 → 確認，或在終端執行：
```bash
xattr -cr /Applications/ABC管理系统.app
```

### Q：前端代碼變更，需要重新安裝嗎？

**不需要。** 設定好 Updater 後，前端代碼變更時，App 啟動會自動檢測新版本並提示更新，使用者無需手動下載 DMG。見[附記：Updater 更新機制說明](#附記updater-更新機制說明)。
