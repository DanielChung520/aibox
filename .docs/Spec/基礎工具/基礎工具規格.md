## 1. 概述

基礎工具是系統聊天功能的核心依賴，提供 AI Agent 無法自行取得的外部資訊（實時網頁資料、天氣）。所有基礎工具透過統一的 `BaseTool` 框架實作，設定集中存放於 ArangoDB `system_params` 集合，由 Rust API Gateway 統一提供讀取端點。

### 1.1 工具清單

| 工具名稱 | 功能 | 提供商 |
|----------|------|--------|
| `web_search` | Web 搜尋（自動降級） | Serper → SerpAPI → ScraperAPI → Google CSE |
| `weather` | 當前天氣查詢 | OpenWeatherMap |
| `forecast` | 天氣預報（1-7天） | OpenWeatherMap |

## 2. 架構設計

```
┌─────────────────────────────┐
│       Frontend (React)       │
│  SystemParams > 基礎工具 Tab  │
└──────────────┬──────────────┘
               │ POST /api/v1/system-params/{key}
               ▼
┌─────────────────────────────┐
│     Rust API Gateway (6500)  │
│  GET /api/v1/web-search/config │
│  GET /api/v1/weather/config    │
│  system_params (ArangoDB)      │
└──────────────┬──────────────┘
               │ httpx GET (Python → Rust)
               ▼
┌─────────────────────────────┐
│  MCP Tools Service (8004)    │
│  tools/web_search/           │
│  tools/weather/             │
│  tools/base.py (BaseTool)    │
└──────────────┬──────────────┘
               │
    ┌──────────┴──────────┐
    ▼                      ▼
External APIs           External APIs
(Serper/SerpAPI/...)   (OpenWeatherMap)
```

### 2.1 設計原則

- **設定集中化**：所有 API Key 存放於 `system_params`，非環境變數
- **Lazy Load**：Python 工具在 `__init__` 時不發起 HTTP 請求，僅在 `execute()` 時讀取設定
- **自動降級**：Web 搜尋按優先級自動切換可用提供商
- **記憶體緩存**：工具結果帶 TTL 緩存，減少外部 API 呼叫

### 3.3 Provider Card 結構

每個 Provider Card 包含：
- 名稱 + 描述 + Switch 啟用開關
- API Key 輸入框（`Input.Password`）
- 額外欄位（如 Google CSE 的 CX）
- 狀態指示燈（綠色=已填寫 / 紅色=待填寫）

### 3.4 資料流

1. `useEffect` → `GET /api/v1/system-params` → 載入所有 `web_search.*` / `weather.*` 參數
2. 使用者切換 Switch 或輸入 → 更新 local state (`values`)
3. 點擊「保存全部」→ `PUT /api/v1/system-params/{key}` 逐一更新

### 5.2 Seed 機制

在 `api/src/db/mod.rs` 中有兩個 seed 函數，服務啟動時自動執行（僅在記錄不存在時寫入）：

- `ensure_websearch_params()` → 建立上網搜尋預設參數
- `ensure_weather_params()` → 建立天氣預設參數

### 6.2 BaseTool 框架

```python
class BaseTool(ABC, Generic[T_in, T_out]):
    @property
    def name(self) -> str: ...
    @property
    def description(self) -> str: ...
    @property
    def version(self) -> str: return "1.0.0"
    @abstractmethod
    async def execute(self, input_data: T_in) -> T_out: ...
```

### 6.3 緩存策略

| 工具 | TTL |
|------|-----|
| `web_search` | 30 分鐘 |
| `weather` | 10 分鐘 |
| `forecast` | 1 小時 |

緩存 Key 格式：`SHA256(prefix|key1=value1|key2=value2|...)` 前 32 字元

### 6.4 Lazy Load 設計

```python
class WebSearchTool(BaseTool):
    def __init__(self, search_service=None):
        # 不在此時創建 WebSearchService，避免發起 HTTP 請求
        self._search_service = search_service

    async def execute(self, input_data):
        # 第一次執行時才初始化
        if self._search_service is None:
            self._search_service = WebSearchService()
        ...
```

## 9. 工具詳細規格

### 9.1 Web Search

#### 輸入 (WebSearchInput)

| 欄位 | 型別 | 預設值 | 說明 |
|------|------|--------|------|
| `query` | string | 必填 | 搜尋查詢字串 |
| `num` | integer | 10 | 結果數量（1-100） |
| `location` | string? | null | 地理位置（如 "Taiwan"） |

#### 輸出 (WebSearchOutput)

| 欄位 | 型別 | 說明 |
|------|------|------|
| `query` | string | 原始查詢 |
| `provider` | string | 實際使用的提供商名稱 |
| `results` | SearchResult[] | 結果列表 |
| `total` | integer | 結果總數 |
| `status` | string | `success` / `failed` |

#### SearchResult 結構

```typescript
interface SearchResult {
  title: string;       // 標題
  link: string;        // 連結
  snippet: string;     // 摘要
  result_type: 'organic' | 'answer_box';  // 結果類型
  position: number;    // 排名
}
```

#### 優先級降級順序

```
Serper → SerpAPI → ScraperAPI → Google CSE
  ↑                                    ↑
  └── 所有 Provider 都失敗 → error ─────┘
```

### 9.2 Weather

#### 輸入 (WeatherInput)

| 欄位 | 型別 | 預設值 | 說明 |
|------|------|--------|------|
| `city` | string? | null | 城市名稱（與 lat/lon 二選一） |
| `lat` | float? | null | 緯度 |
| `lon` | float? | null | 經度 |
| `units` | string | `metric` | `metric`（°C）或 `imperial`（°F） |

#### 輸出 (WeatherOutput)

| 欄位 | 型別 | 說明 |
|------|------|------|
| `city` | string | 城市名稱 |
| `country` | string | 國家代碼 |
| `temperature` | float | 溫度 |
| `feels_like` | float | 體感溫度 |
| `humidity` | int | 濕度（%） |
| `pressure` | int | 氣壓（hPa） |
| `description` | string | 天氣描述 |
| `icon` | string | OpenWeatherMap icon code |
| `wind_speed` | float | 風速 |
| `wind_direction` | int | 風向（度數） |
| `visibility` | int? | 能見度（米） |
| `timestamp` | float | 資料時間戳 |

### 9.3 Forecast

#### 輸入 (ForecastInput)

| 欄位 | 型別 | 預設值 | 說明 |
|------|------|--------|------|
| `city` | string? | null | 城市名稱 |
| `lat` | float? | null | 緯度 |
| `lon` | float? | null | 經度 |
| `days` | int | 3 | 預報天數（1-7） |
| `hourly` | bool | false | 是否包含小時級別預報 |
| `units` | string | `metric` | 溫度單位 |

#### 輸出 (ForecastOutput)

```typescript
interface ForecastOutput {
  city: string;
  country: string;
  forecasts: ForecastItem[];
}

interface ForecastItem {
  date: string;           // 日期 (YYYY-MM-DD)
  temperature: float;     // 當日溫度
  min_temp: float;
  max_temp: float;
  description: string;
  icon: string;
  humidity: int;
  wind_speed: float;
  precipitation: float?;   // 降水量（mm）
  hourly?: HourlyForecast[];
  timestamp: float;
}
```

## 10. 錯誤處理

| 錯誤類別 | 發生時機 | 處理方式 |
|----------|----------|----------|
| `ToolValidationError` | 輸入參數驗證失敗 | 立即回傳，不呼叫外部 API |
| `ToolExecutionError` | API Key 未設定 / 外部 API 錯誤 | 往上層拋 |
| `RuntimeError` (WebSearchService) | 無任何可用 Provider | 初始化時拋出，工具無法使用 |

## 11. 安全性考量

- API Key 存放於 ArangoDB `system_params`，非程式碼或環境變數
- 前端 `Input.Password` 遮蔽顯示
- Python 工具透過 Rust API 讀取設定，不直接暴露 Key
- Rust API 僅提供讀取（GET），寫入（PUT）需 JWT 認證
