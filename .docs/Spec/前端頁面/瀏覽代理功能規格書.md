# 瀏覽代理 (Browse Agent) 功能規格書

## 1. 概述

### 功能目標
以卡片式展示已登記的 BPA (Business Process Agent) Agent，支援分類瀏覽、執行、編輯、刪除等操作。

### 資料來源
- **BPA Service** (port 8005) - Workflow 定義
- 現有 Workflow: `user_onboarding`, `data_backup`

### 2.2 元件說明

#### Tab 分類
- 顯示所有 Agent 分類
- 點擊切換篩選對應分類的 Agent
- 分類來自：workspace.browse-agent 下的子功能

#### Agent 卡片 (AgentCard)
| 欄位 | 說明 |
|------|------|
| Icon | 40x40 圖標 |
| Name | Agent 名稱 |
| Description | 描述文字 |
| Status | 狀態標籤 (在線/維修中/審查中/已作廢) |
| Usage Count | 使用次數 |
| Actions | 收藏 ❤️、更多 ⋮ (編輯/刪除) |
| Chat Button | 啟動對話 |

#### 狀態顏色
| 狀態 | 顏色 |
|------|------|
| 在線 (online) | 綠色 |
| 維修中 (maintenance) | 黃色 |
| 審查中 (registering) | 橙色 |
| 已作廢 (deprecated) | 紅色 |

## 3. API 規格

### 3.1 現有 API (BPA Service)

| 方法 | 端點 | 說明 |
|------|------|------|
| GET | `/workflows` | 取得所有 workflow |
| GET | `/workflows/{id}` | 取得特定 workflow |
| POST | `/execute` | 執行 workflow |
| POST | `/execute-async` | 非同步執行 |

### 3.2 需要新增的 API

| 方法 | 端點 | 說明 |
|------|------|------|
| GET | `/api/v1/agents` | 取得所有已註冊 Agent |
| POST | `/api/v1/agents` | 註冊新 Agent |
| PUT | `/api/v1/agents/{id}` | 更新 Agent |
| DELETE | `/api/v1/agents/{id}` | 刪除 Agent |
| POST | `/api/v1/agents/{id}/execute` | 執行 Agent |
| GET | `/api/v1/agent-groups` | 取得 Agent 分類 |

### 3.3 資料模型

```typescript
interface Agent {
  _key: string;
  code: string;              // Agent 代碼 (如 "user-onboarding")
  name: string;              // 顯示名稱
  description: string;       // 描述
  icon: string;              // 圖標名稱
  status: 'registering' | 'online' | 'maintenance' | 'deprecated';
  group_key: string;        // 所屬分類
  workflow_id: string;      // 對應的 workflow ID
  endpoint: string;         // 端點 URL
  protocol: 'http' | 'mcp'; // 協議類型
  usage_count: number;       // 使用次數
  is_favorite: boolean;     // 是否收藏
  created_at: string;
  updated_at: string;
}

interface AgentGroup {
  _key: string;
  code: string;
  name: string;
  icon: string;
  sort_order: number;
}
```

## 4. 資料儲存

### 4.1 ArangoDB Collections

```
agents          - Agent 註冊資料
agent_groups   - Agent 分類
```

### 4.2 分類定義

Agent Group 存放於 `agent_groups` 集合，與現有 `functions` 分離（functions 用於選單權限，agent_groups 用於 Agent 分類）。
